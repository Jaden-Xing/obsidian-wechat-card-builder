# 微信文章转卡片笔记

将微信公众号文章转为结构化 Obsidian 卡片笔记：抓取 → 提炼核心观点 / 详细内容 / 我的思考 → 按统一模板写入 vault。

## 配置

- 卡片目录：`D:/06knowleagebase/01wechat-kb/`
- 文件名：`<YYYY-MM-DD>-<标题>.md`（标题中 `\ / : * ? " < > |` 等 Windows 非法字符替换为全角或删除）

## 工作流程

### URL 模式

1. **抓取文章内容**：用 WebFetch 抓取 `mp.weixin.qq.com` 链接，提取以下信息：
   - 文章标题、公众号名称、作者姓名、发布时间（精确到日）、发布地点
   - 文章完整正文（逐段保留，不要遗漏）
   - 文章中图片的描述和图中文字（如有）
   - 文章留言 / 评论（如有）
   - 微信文章图片和留言常为动态加载（JavaScript / API），WebFetch 可能取不到；确实没有就注明，不要无限重试。
2. **分析提炼内容**：
   - **核心观点**：3-6 条，每条 10-25 字，冒号分隔主题与描述
   - **详细内容**：按原文逻辑分段，每段 2-3 句话概括
   - **我的思考**：联系实际工作，指出可取与局限，一段一义 ≤ 三句，语气像真实笔记主人（可吐槽 / 质疑）
   - **标签**：自动推荐 2-5 个，反映内容实质；默认追加 `wechat`
   - **原文内容**：保留抓取到的完整原文全文
3. **写入文件**：生成 `<YYYY-MM-DD>-<标题>.md`，格式见 `references/card-template.md`。Obsidian 正在运行该 vault 时会自动索引。

### 文章分析模式

触发：本地文件路径 / 文件夹路径 + 关键词「文章分析」。

1. **定位目标文件**：文件路径 → 直接分析该文件；文件夹路径 → 递归扫描目录下所有 `.md`。
2. **解析卡片**：按一级标题锚点读取 `# 核心观点` / `# 详细内容` / `# 我的思考` / `# 原文内容`，以及 frontmatter 的 `tags` / `source` / `title` / `author` / `published_at` 等属性。
3. **分析内容**：
   - 提炼新增标签，与现有标签去重合并
   - 若 `# 我的思考` 为空或较浅，补充思考
4. **更新卡片**：保留已有 frontmatter 属性，只做增量修改：
   - `tags` 合并去重（默认保留 `wechat` 等原有标签）
   - `# 我的思考` 中插入新内容（用 `---` 分隔）
   - `# 原文内容` 原样不动，绝不改写

## 卡片格式（通用规则）

卡片遵循 **Obsidian Flavored Markdown** 规范（详见 `obsidian-markdown` skill），格式对齐要点如下。

### Frontmatter 属性

- **`tags`**：YAML 块列表（非内联数组）；默认含 `wechat`；支持嵌套标签 `wechat/产品`、`product/prd`
- **`title`** / **`source`** / **`description`**：文本
- **`author`**：用 wikilink 指向 vault 内作者笔记，如 `author: "[[张佳]]"`，便于图谱串联
- **`published_at`**：Obsidian「日期与时间」属性，ISO 8601，如 `2026-07-03T12:10:00+08:00`
- **`aliases`**（可选）：作者 / 文章别名，提升链接与检索友好度
- 抓取不到的字段（author / location 等）可省略

### Obsidian 专属语法

- **内部链接**：vault 内其他卡片 / 作者用 `[[Note]]` 或 `[[Note|显示文本]]`；外部链接用标准 `[文本](url)`
- **标注（callout）**：重要提示可用 `> [!note]` / `> [!tip]` / `> [!warning]` 等（详见 `obsidian-markdown`）
- **嵌入**：本地图片用 `![[image.png]]`，外链图片用 `![alt](url)`
- **高亮**：`==重点==` 标记关键词

### 标题锚点（必须保留）

正文四个一级标题 `# 核心观点` / `# 详细内容` / `# 我的思考` / `# 原文内容` 必须保留，便于文章分析模式按标题解析。

### 原文内容铁律

`# 原文内容` 直接粘贴原始文章全文，保持原文自身格式（小标题、分割线 `---`、图片链接、表格等）；**不要给原文每行加 `>` 前缀，不要添加公众号 / 作者 / 时间摘要行，不要添加说明行，也不要用 callout 包裹**。

标签不要过于宽泛（如「文章」「笔记」），应使用能反映内容实质的标签；「我的思考」不要写成总结，应是有个人视角的独立评论。

完整模板见 `references/card-template.md`。
